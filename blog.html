1. What is DOM and what is the purpose of DOM?

The Document Object Model (DOM) represents the page so that programs can change the document structure, style, and content. The DOM is represented as a type of data structure called a tree. Every object in the DOM is hierarchically under another object, and any object can have multiple children but only one parent. The purpose of document object model is that it provides an interface to create elements, navigate them and to add, remove, and modify elements of a website.

2. How will you select HTML elements using DOM? Name the DOM methods.

We can select html elements by the following ways:
document.getElementById() // by ID
document.getElementsByClassName()// by Class name
document.getElementsByTagName() // by tag name
document.querySelector(selector) // by CSS selector
document.querySelectorAll(selector)

Some of the DOM methods are:
1. createElement: It is used to create an elements.
2. appendChild: It is used to add an elements.
3. removeChild: It is used to reomove an elements.
4. getAttribute: it is used to access the value of any attribute of an element.
5. setAttribute: Assigning a new value to an existing attribute is done using setAttribute.
6. innerHTML: it is used to access & write content inside HTML elements.

3. What is event bubble?

Event bubble is that when an event happens on an element, it first runs the handlers on it, then on its parent, then all the way up on other ancestors. We can see the flow in the picture. We can stop bubble by using the method event.stopPropagation().

4. What is a callback function and why will you use it?

A callback function is a function passed into another function as an argument, which is then invoked inside the outer function to complete some kind of routine or action. When the main function is finished, it calls the callback function to provide a result. Callbacks create orders in code so that they can wait for a response from an API request or a timed action before executing another piece of code. Callbacks allow us to continue executing code while the operation is being executed in the background. Once the operation has been completed, the callback function is called with the result of the operation.
